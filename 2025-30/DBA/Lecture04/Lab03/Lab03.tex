\section{Introduction}
In this lab, we will explore another DBA tool, this time oriented towards monitoring the state of our databases—particularly the behavior and performance of running queries. We will use a single Python or Bash script to feed data into a PostgreSQL table online, simulating a streaming input. We will install and deploy a dashboard panel using Grafana to monitor the ongoing data. We will assume that you already have a PostgreSQL server and a Python environment running on your machine.


\section{Grafana + PostgreSQL}
You will watch the following videos to see how to connect Grafana with a table in PostgreSQL that stores sensor data. The videos are \href{https://drive.google.com/file/d/1NpSFTgN7s5IwQjOtmUd-gfDtW3V7F4Rt/view?usp=sharing}{this} and \href{https://drive.google.com/file/d/1nxkS3YTr_MTg1tvGLcMb8NuLeJ245NBv/view?usp=sharing}{this}. You can find the code to create the table as follows:

\begin{minted}
[tabsize=4, obeytabs, frame=lines, framesep=2mm, baselinestretch=1.2, bgcolor=LightGray, fontsize=\footnotesize]{sql}
CREATE TABLE IF NOT EXISTS public.sensor
(
    oid bigint NOT NULL,
    lat real NOT NULL,
    lon real NOT NULL,
    tim timestamp with time zone NOT NULL,
    CONSTRAINT sensor_pkey PRIMARY KEY (tim)
)
\end{minted}
